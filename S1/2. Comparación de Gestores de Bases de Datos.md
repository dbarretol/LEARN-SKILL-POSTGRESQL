## Separata: Comparación de Gestores de Bases de Datos (PostgreSQL, SQL Server, MySQL)

Esta sección detalla la comparación entre PostgreSQL, SQL Server y MySQL, cubriendo sus aspectos de licencia, desarrollo y características técnicas clave.

### A. Aspectos de Licencia y Desarrollo

| Característica | PostgreSQL (PGRE) | SQL Server | MySQL |
| :--- | :--- | :--- | :--- |
| **Licencia** | **Open Source** (licencia libre). | Propietario de Microsoft; cobra un costo, dependiendo de la versión. | **Open Source** con opciones comerciales. |
| **Desarrollador** | Postgre Global Development Group. | Microsoft. | Oracle Corporation. |

### B. Características Técnicas

La siguiente tabla resume las diferencias técnicas y el rendimiento entre los tres gestores de bases de datos:

| Característica | PostgreSQL (PGRE) | SQL Server | MySQL |
| :--- | :--- | :--- | :--- |
| **Modelo de Datos** | Relacional con **soporte completo de objetos**. | Relacional. | Relacional. |
| **Soporte JSON y NoSQL** | **Soporte avanzado** en consultas, índices y funciones JSON. | Limitado pero funcional. | Soporte básico, menos eficiente que PostgreSQL. |
| **Rendimiento (Lectura)** | Muy bueno para **consultas complejas**. | Excelente entorno corporativo. | Muy rápido para **lecturas simples**. |
| **Rendimiento (Escritura)** | Muy bueno. | Excelente en escenarios controlados. | Bueno en operaciones simples. |
| **Escalabilidad** | Muy buena para escalabilidad **horizontal o vertical**,. | Alta escalabilidad vertical. | Moderada. |
| **Lenguaje SQL** | **Altamente estándar** y extendible. | Transact-SQL. | SQL con limitaciones frente al estándar. |
| **Transacciones ACID** | Completo. | Completo. | Completo, pero **depende del motor de almacenamiento**. |
| **Índices Avanzados** | Utiliza índices avanzados como **GIS, GIN y BRM**. | Índices tradicionales y *column store*. | Índices básicos. |
| **Particionado de Tablas** | Soportado desde versiones actuales. | Eficiente y maduro. | Limitado. |
| **Seguridad** | **Robustez** con roles y políticas de autenticación. | Más robusta con integración con **Active Directory**. | Básica. |
| **Uso Ideal** | Aplicaciones complejas, análisis y **GIS** (Sistemas de Información Geográfica). | Grandes empresas. | Aplicaciones web ligeras. |

En resumen, **PostgreSQL** es el sistema más recomendado para aplicaciones complejas que requieren análisis avanzado y el manejo de datos con soporte completo de objetos, mientras que **MySQL** se destaca por su velocidad en lecturas simples para aplicaciones web ligeras. **SQL Server** es la opción ideal en grandes entornos corporativos debido a su robustez y seguridad avanzada con integración de directorio activo,.
