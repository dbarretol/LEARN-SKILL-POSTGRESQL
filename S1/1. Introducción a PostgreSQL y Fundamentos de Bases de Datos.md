## Separata: Introducción a PostgreSQL y Fundamentos de Bases de Datos

### I. Conceptos Básicos y Roles de PostgreSQL

El material discutido introduce el **módulo de gestión de bases de datos utilizando PostgreSQL**.

Una **base de datos** se define como un conjunto de datos que se almacenan y pueden consultarse fácilmente.

**PostgreSQL** forma parte de un **Sistema de Gestión de Base de Datos (SGDB)**, cuya función es permitir la administración y consulta de dichas bases de datos, de forma similar a otros gestores como SQL Server, MySQL u Oracle.

### II. Componentes Fundamentales de Bases de Datos Relacionales

Las bases de datos, en general, están siempre conformadas por los siguientes componentes clave:

*   **Tabla:** Es la estructura fundamental que almacena los datos, organizados en filas y columnas.
*   **Campos o Columnas:** Son los atributos de la tabla donde se define el tipo de datos que se almacenarán, lo cual es esencial para guardar la información correctamente.
*   **Registro o Filas:** Representan un conjunto de datos relacionados que conforman una línea dentro de una tabla.
*   **Clave Primaria (*Primary Key*):** Es una combinación de campos que tiene la función de identificar de manera única a cada registro dentro de una tabla, siendo crucial para la identificación.
*   **Claves Foráneas (*Foreign Keys*):** Son campos que se utilizan para referenciar la clave primaria de otra tabla, permitiendo así establecer relaciones entre tablas y gestionar el sistema.
*   **Consultas (*Queries*):** Son instrucciones que forman parte esencial de una base de datos y se utilizan para buscar, actualizar, insertar, agregar o eliminar los registros que se encuentran almacenados.

#### A. Lenguajes utilizados en PostgreSQL

PostgreSQL, al igual que MySQL, utiliza el **lenguaje estándar SQL** (*Structured Query Language*). Este lenguaje de consulta estructurado es manejado por la mayoría de los gestores de bases de datos.

Además del SQL estándar, PostgreSQL también maneja su **propio lenguaje de programación, conocido como PL/pgSQL (LP Podre)**, que domina la estructura de la base de datos.

### III. Conceptos Avanzados de Gestión de Datos

Para mantener la integridad y eficiencia de una base de datos, se utilizan conceptos avanzados:

*   **Relación:** Es el vínculo lógico que se establece entre tablas, fundamental para mantener la integridad de los datos, apoyado por el uso de claves foráneas.
*   **Índices:** Se trata de una estructura diseñada para **mejorar la velocidad de las consultas** ejecutadas sobre una o más columnas de la tabla.
*   **Normalización:** Es un proceso muy importante y básico en las bases de datos relacionales, que consiste en organizar los datos para **reducir la redundancia y las dependencias** entre ellos.
*   **Transacción:** Es un conjunto de operaciones que se ejecutan como una **unidad invisible**.
*   **Backups:** Son las copias de seguridad de la base de datos, utilizadas para la **recuperación ante cualquier fallo** o desastre que pueda ocurrir con la data.
