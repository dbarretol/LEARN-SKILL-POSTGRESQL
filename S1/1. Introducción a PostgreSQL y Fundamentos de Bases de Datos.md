## Separata: Introducción a PostgreSQL y Fundamentos de Bases de Datos

### I. Conceptos Básicos

Este módulo introduce la gestión de bases de datos utilizando **PostgreSQL**.

- **Base de Datos (BD):**  
  Conjunto de datos almacenados de forma organizada para su uso y consulta posterior.

- **SGBD (Sistema de Gestión de Bases de Datos):**  
  Software que permite la administración, protección y consulta de bases de datos.  
  - **PostgreSQL:** Sistema de base de datos relacional de objetos (ORDBMS), similar a otros gestores como **SQL Server**, **MySQL** u **Oracle**.

---

### II. Componentes Fundamentales de Bases de Datos Relacionales

Las bases de datos relacionales se estructuran a partir de los siguientes componentes clave:

- **Tabla:**  
  Estructura fundamental donde se almacenan los datos, organizados en filas y columnas.

- **Campos (Columnas):**  
  Atributos de la tabla. En cada columna se define un **tipo de dato** (texto, número, fecha, etc.), que determina qué clase de información se puede almacenar.

- **Registros (Filas):**  
  Cada fila representa una entrada completa de datos relacionados dentro de una tabla.

- **Clave Primaria (*Primary Key – PK*):**  
  Columna (o conjunto de columnas) que identifica de manera **única** a cada registro.  
  - No puede contener valores repetidos ni valores nulos.

- **Clave Foránea (*Foreign Key – FK*):**  
  Columna que referencia la clave primaria de otra tabla, permitiendo establecer **relaciones** entre tablas y mantener la integridad referencial.

- **Consultas (*Queries*):**  
  Instrucciones utilizadas para interactuar con la base de datos:  
  - **Seleccionar** datos (`SELECT`)  
  - **Insertar** nuevos registros (`INSERT`)  
  - **Actualizar** datos existentes (`UPDATE`)  
  - **Eliminar** registros (`DELETE`)

#### A. Lenguajes utilizados en PostgreSQL

1. **SQL (*Structured Query Language*):**  
   Lenguaje estándar utilizado por la mayoría de los SGBD para definir estructuras y realizar consultas y mantenimiento de datos.

2. **PL/pgSQL (*Procedural Language/PostgreSQL*):**  
   Lenguaje de programación propio de PostgreSQL que permite:
   - Crear funciones y procedimientos almacenados.
   - Implementar disparadores (*triggers*).
   - Controlar la lógica de negocio dentro de la base de datos (condiciones, bucles, variables, etc.).

### III. Conceptos Avanzados de Gestión de Datos

Para garantizar la integridad, eficiencia y seguridad de la información, se aplican los siguientes conceptos:

- **Relación:**  
  Vínculo lógico entre tablas (generalmente mediante claves foráneas).  
  Es esencial para mantener la consistencia de los datos en un modelo relacional.

- **Índices:**  
  Estructuras de datos auxiliares que permiten **acelerar la búsqueda y recuperación** de información.  
  Mejoran el rendimiento de las consultas a costa de ocupar espacio adicional y requerir mantenimiento en escrituras.

- **Normalización:**  
  Proceso de organización de los datos en tablas y columnas para:  
  - **Reducir la redundancia** (evitar datos duplicados).  
  - **Mejorar la integridad** de las dependencias entre datos.  
  Se apoya en reglas llamadas **formas normales**.

- **Transacción:**  
  Conjunto de operaciones que se ejecutan como una **unidad indivisible** (*todo o nada*).  
  Si alguna operación falla, se deshacen todas las demás para mantener la consistencia de la base de datos (propiedad de atomicidad en ACID).

- **Backups (Copias de seguridad):**  
  Copias de los datos y/o la estructura de la base de datos que permiten:  
  - Recuperarse ante fallos del sistema, errores humanos o desastres.  
  - Restaurar la información a un punto anterior en el tiempo.
