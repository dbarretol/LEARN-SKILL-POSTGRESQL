## Separata: Tipos de Datos en PostgreSQL

Esta sección describe los tipos de datos fundamentales y avanzados en **PostgreSQL**, esenciales para definir correctamente los atributos de las tablas.

---

### A. Tipos Numéricos

PostgreSQL ofrece varios tipos para manejar números enteros y decimales, con distintos rangos y niveles de precisión:

* **INTEGER, SMALLINT, BIGINT:**  
  Se utilizan para almacenar **números enteros** de diferentes tamaños.
  * `SMALLINT`: enteros pequeños (2 bytes).  
  * `INTEGER` (o `INT`): enteros estándar (4 bytes).  
  * `BIGINT`: enteros grandes (8 bytes).

* **DECIMAL(p, s) y NUMERIC(p, s):**  
  Se usan para **números decimales con precisión exacta** (por ejemplo, montos de dinero).  
  En PostgreSQL, `DECIMAL` y `NUMERIC` son equivalentes; ambos permiten especificar precisión (`p`) y escala (`s`).

* **REAL y DOUBLE PRECISION:**  
  Tipos de **números en punto flotante** (aproximados).
  * `REAL`: precisión simple, 4 bytes.  
  * `DOUBLE PRECISION`: precisión doble, 8 bytes.

---

### B. Tipos de Cadena de Texto

Para datos alfanuméricos, PostgreSQL ofrece:

* **VARCHAR(n):**  
  Cadena de texto de **longitud variable** con límite máximo de *n* caracteres.

* **CHAR(n):**  
  Cadena de texto de **longitud fija** de *n* caracteres.  
  Siempre reserva el espacio de *n* caracteres, incluso si se almacenan menos.

* **TEXT:**  
  Cadena de texto de **longitud prácticamente ilimitada**.  
  Ideal para textos largos (descripciones, comentarios) y también se usa con frecuencia para almacenar JSON en formato texto.

---

### C. Tipos Lógicos, de Fecha y Avanzados

#### 1. Tipo Lógico

* **BOOLEAN:**  
  Representa valores de verdad: `TRUE`, `FALSE` o `NULL`.

#### 2. Tipos de Fecha y Hora

* **DATE:**  
  Almacena una fecha (año, mes, día).

* **TIME [WITHOUT TIME ZONE]:**  
  Almacena solo la hora (sin fecha).

* **TIMESTAMP WITHOUT TIME ZONE:**  
  Almacena fecha y hora, sin zona horaria.

* **TIMESTAMP WITH TIME ZONE (timestamptz):**  
  Almacena fecha y hora ajustadas a una **zona horaria**.  
  PostgreSQL convierte internamente según la configuración de zona horaria del servidor/cliente.

* **INTERVAL:**  
  Representa un **periodo de tiempo o duración** (por ejemplo, “3 días”, “2 horas 30 minutos”).

#### 3. Tipos Estructurados y NoSQL

* **JSON:**  
  Almacena datos JSON como **texto**. Conserva el formato original de la cadena.

* **JSONB:**  
  Tipo nativo de PostgreSQL que almacena JSON en **formato binario**.  
  Ventajas: más eficiente para **búsquedas, filtros e indexación**, a costa de usar algo más de espacio y perder el formato textual original.

* **ARRAY:**  
  Permite almacenar una **lista de valores del mismo tipo** en una sola columna.  
  Ejemplo: `integer[]`, `text[]`.

* **HSTORE:**  
  Tipo que almacena un **conjunto de pares clave–valor** dentro de un solo campo, útil para datos semiestructurados.

---

En resumen, PostgreSQL ofrece un conjunto muy completo de tipos de datos, desde tipos numéricos y de texto básicos hasta estructuras avanzadas como `JSONB`, `ARRAY` y `HSTORE`, lo que lo hace especialmente potente para trabajar tanto con datos relacionales tradicionales como con datos semiestructurados o de estilo NoSQL.
