## Separata: Tipos de Datos en PostgreSQL

Esta sección describe los tipos de datos fundamentales y avanzados utilizados en PostgreSQL, esenciales para la definición de los atributos de las tablas.

### A. Tipos Numéricos

PostgreSQL ofrece una variedad de tipos de datos para manejar números enteros y decimales, asegurando la precisión y el rango necesarios:

*   **INTEGER, SMALLINT, BIGINT:** Se utilizan para almacenar **números enteros** de diferentes tamaños. El tipo `SMALLINT` almacena números enteros pequeños, mientras que `BIGINT` almacena números enteros grandes.
*   **DECIMAL y NUMERIC:** Ambos se usan para **números decimales con precisión exacta**. El tipo **`NUMERIC`** se destaca por ofrecer una **mayor precisión** en comparación con `DECIMAL`.
*   **REAL y DOUBLE PRECISION:** Estos son tipos de **números de punto flotante**.
    *   `REAL` es de precisión simple y ocupa 4 bytes.
    *   `DOUBLE PRECISION` es de precisión doble y ocupa 8 bytes.

### B. Tipos de Cadena de Texto

Para almacenar datos alfanuméricos, PostgreSQL utiliza los siguientes tipos de cadena:

*   **VARCHAR(n):** Almacena una **cadena de texto de longitud variable** con un límite máximo de *n* caracteres.
*   **CHAR(n):** Almacena una **cadena de texto de longitud fija** de *n* caracteres. Una característica clave de `CHAR(n)` es que, incluso si el campo no contiene datos, seguirá ocupando el espacio de *n* caracteres.
*   **TEXT:** Se utiliza para **cadenas de texto sin límites de longitud**. Es ideal para almacenar texto largo, y también es adecuado para guardar datos JSON.

### C. Tipos Lógicos, de Fecha y Avanzados

PostgreSQL también maneja valores de verdad, fechas, y estructuras de datos más complejas:

*   **BOOLEAN:** Se utiliza para la **representación de valores de verdad**. Sus posibles valores son `True`, `False` o `Null`.
*   **Tipos de Fecha y Hora:**
    *   **DATE:** Almacena la fecha (año, mes, día).
    *   **TIME:** Almacena la hora sin fecha.
    *   **TIMESTAMP WITHOUT TIME ZONE:** Almacena fecha y hora, sin incluir la zona horaria.
    *   **TIMESTAMP WITH TIME ZONE:** Almacena fecha y hora, e incluye la zona horaria. Este tipo devuelve la fecha horaria correspondiente al país.
    *   **INTERVAL:** Representa un **periodo de tiempo** o una duración entre dos fechas u horas.
*   **Tipos Estructurados y NoSQL:**
    *   **JSON:** Almacena datos JSON en formato de texto.
    *   **JSONB:** Es un tipo de dato propio de PostgreSQL (PGRE). Almacena datos JSON en **formato binario**, haciéndolo **más eficiente para búsquedas y operaciones**, aunque requiere más espacio de almacenamiento.
    *   **ARRAY:** Permite almacenar una **lista de valores del mismo tipo** en un solo campo.
    *   **HSTORE:** Se utiliza para almacenar un **conjunto de pares clave-valor** en un único campo.

En esencia, PostgreSQL proporciona herramientas robustas para definir con precisión la naturaleza de los datos, desde los tipos básicos de texto y números hasta estructuras más complejas como JSONB, que es más avanzado en el manejo de estructuras de datos que otros gestores.
