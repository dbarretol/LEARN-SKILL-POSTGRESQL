## Separata: Instalación y Configuración Inicial de PostgreSQL

Esta sección describe el proceso de instalación de **PostgreSQL**, las herramientas esenciales para interactuar con el gestor de bases de datos y los primeros pasos con comandos SQL básicos.

---

### A. Proceso de Instalación

PostgreSQL puede instalarse en los principales sistemas operativos:

- **Windows:**  
  Mediante el instalador oficial descargado desde:  
  `https://www.postgresql.org/download/windows/`

- **Linux (distribuciones GNU/Linux):**  
  Usualmente disponible en los repositorios oficiales de cada distribución o mediante contenedores (por ejemplo, Docker).  
  Ejemplo en Debian/Ubuntu:
  ```bash
  sudo apt update
  sudo apt install postgresql postgresql-contrib
  ```

- **macOS:**  
  Puede instalarse con el instalador oficial o mediante Homebrew:
  ```bash
  brew install postgresql
  ```

El **puerto por defecto** utilizado por PostgreSQL es el **5432** (configurable).

#### Configuración inicial típica

Tras la instalación, los pasos iniciales más comunes son:

1. Abrir **pgAdmin 4**.
2. Conectarse al servidor PostgreSQL.
3. Crear una **base de datos**.
4. Crear un **rol de usuario** (con o sin privilegios de superusuario).
5. Ejecutar consultas utilizando el **Query Tool** de pgAdmin o la consola `psql`.

---

### B. Herramientas de Interacción

Las dos herramientas principales para administrar y trabajar con PostgreSQL son:

- **pgAdmin 4:**  
  Interfaz gráfica de administración que se ejecuta en un navegador web. Permite:
  - Gestionar servidores, bases de datos, esquemas, tablas, roles, etc.
  - Ejecutar consultas con el **Query Tool**.

- **psql:**  
  Consola de comandos oficial de PostgreSQL. Permite:
  - Conectarse a bases de datos.
  - Ejecutar comandos SQL y comandos internos de `psql`.
  
  Ejemplo de conexión:
  ```bash
  psql -U postgres -h localhost -p 5432
  ```

---

### C. Primeros Pasos y Comandos Básicos

Una vez establecida la conexión al servidor y a una base de datos:

- En **pgAdmin 4**, las consultas se ejecutan desde el **Query Tool**.
- En **psql**, se escriben directamente en la consola.

#### Comandos SQL básicos (DML)

Estos comandos son esenciales para la manipulación de datos:

- **`SELECT`**: Recuperar datos desde una o más tablas.
- **`INSERT`**: Insertar nuevos registros en una tabla.
- **`UPDATE`**: Modificar datos existentes.
- **`DELETE`**: Eliminar registros existentes.

Ejemplo rápido:
```sql
SELECT * FROM nombre_tabla;
INSERT INTO nombre_tabla (col1, col2) VALUES ('valor1', 'valor2');
```

#### Bases de datos predeterminadas

Al iniciar el servidor, es posible visualizar las bases de datos existentes con:

```sql
SELECT * FROM pg_database;
```

Entre ellas se encuentran:

- **`postgres`**: Base de datos predeterminada para uso general.
- **`template1`**: Plantilla principal utilizada para crear nuevas bases de datos.
- **`template0`**: Plantilla mínima, útil para crear bases de datos sin configuraciones adicionales.

Estas bases de datos no deben modificarse sin un propósito claro, ya que son fundamentales para el funcionamiento de PostgreSQL.
